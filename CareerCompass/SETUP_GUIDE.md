# دليل التشغيل السريع - CareerCompass

## 🚀 البدء السريع

### الخطوة 1: تثبيت Node.js

تأكد من تثبيت Node.js على جهازك:
1. قم بتحميل Node.js من https://nodejs.org
2. اختر النسخة LTS (الموصى بها)
3. قم بتثبيتها واتبع التعليمات

للتحقق من التثبيت، افتح Terminal وأدخل:
```bash
node --version
npm --version
```

### الخطوة 2: فك الضغط عن المشروع

قم بفك ضغط مجلد `CareerCompass` إلى موقع على جهازك.

### الخطوة 3: فتح Terminal في مجلد المشروع

#### على Windows:
1. افتح مجلد المشروع في File Explorer
2. اضغط Shift + Right Click
3. اختر "Open PowerShell window here" أو "Open Command window here"

#### على Mac/Linux:
1. افتح Terminal
2. استخدم أمر `cd` للانتقال لمجلد المشروع:
```bash
cd /path/to/CareerCompass
```

### الخطوة 4: تثبيت المكتبات المطلوبة

في Terminal، أدخل:
```bash
npm install
```

سيستغرق هذا بضع دقائق لتحميل جميع المكتبات المطلوبة.

### الخطوة 5: تشغيل التطبيق

بعد انتهاء التثبيت، أدخل:
```bash
npm run dev
```

### الخطوة 6: فتح المتصفح

سيظهر رابط مثل:
```
Local: http://localhost:5173
```

افتح هذا الرابط في المتصفح (Chrome، Firefox، Safari، Edge)

## 🎯 استخدام النظام

### 1. الصفحة الرئيسية
- اقرأ عن النظام ومميزاته
- شاهد الجامعات المتاحة
- اضغط "ابدأ الآن مجاناً" أو "إنشاء حساب"

### 2. إنشاء حساب
املأ البيانات:
- الاسم الكامل
- البريد الإلكتروني
- كلمة المرور
- **الصف الدراسي** (9، 10، 11، أو 12)

**مهم**: اختيار الصف يؤثر على نوع النتائج:
- الصفوف 9-11: توصية بالمجال + خطة تطوير
- الصف 12: توصيات مباشرة بالتخصصات الجامعية

### 3. لوحة التحكم
بعد تسجيل الدخول، ستجد:
- **بدء الاختبار**: 48 سؤالاً (10-15 دقيقة)
- **إدخال الدرجات**: درجاتك في 4 مواد
- **استكشف الجامعات**: دليل الجامعات الأردنية

### 4. الاختبار
- 48 سؤالاً موزعة على 3 أبعاد
- مقياس من 1-5 (لا أوافق بشدة → أوافق بشدة)
- كن صادقاً مع نفسك
- يمكنك الرجوع وتعديل الإجابات
- لا توجد إجابات صحيحة أو خاطئة

### 5. إدخال الدرجات (اختياري لكن موصى به)
أدخل درجاتك في:
- الرياضيات
- العلوم
- اللغة العربية
- اللغة الإنجليزية

هذا يساعد في تقديم توصيات أدق.

### 6. النتائج

#### للصفوف 9-11:
- **المجال المقترح**: (صحي، هندسي، تقني، إداري، فنون)
- **نسبة التوافق**: مع كل مجال
- **نقاط القوة**: أفضل 5 مهارات لديك
- **خطة التطوير**: نقاط ضعف مع نصائح للتحسين

#### للصف 12:
- **التخصصات المقترحة**: قائمة بالتخصصات الجامعية
- **ترتيب حسب التوافق**: الأنسب أولاً
- **التوافق مع المجالات**: رسم بياني
- **نقاط القوة**: مهاراتك الأفضل

### 7. دليل الجامعات
- ابحث عن جامعة معينة
- فلتر حسب: النوع (حكومي/خاص)، الموقع
- شاهد التخصصات المتاحة في كل جامعة

## 🛠 حل المشاكل الشائعة

### المشكلة: "npm: command not found"
**الحل**: لم يتم تثبيت Node.js. ارجع للخطوة 1.

### المشكلة: أخطاء أثناء `npm install`
**الحل**: 
```bash
# احذف مجلد node_modules
rm -rf node_modules
# احذف package-lock.json
rm package-lock.json
# أعد التثبيت
npm install
```

### المشكلة: "Port 5173 already in use"
**الحل**: هناك تطبيق آخر يستخدم نفس المنفذ.
```bash
# استخدم منفذ آخر
npm run dev -- --port 3000
```

### المشكلة: الصفحة فارغة أو بيضاء
**الحل**:
1. افتح Developer Tools (F12)
2. تحقق من Console للأخطاء
3. تأكد من أن جميع الملفات موجودة
4. أعد تشغيل `npm run dev`

### المشكلة: النصوص غير عربية أو غير واضحة
**الحل**: 
- تأكد من اتصالك بالإنترنت (للخطوط من Google Fonts)
- امسح الكاش (Ctrl+Shift+Delete)
- أعد تحميل الصفحة (Ctrl+R)

## 📱 اختبار على الموبايل

1. تأكد أن جهازك والموبايل على نفس الشبكة (WiFi)
2. ابحث عن عنوان IP جهازك:
   - Windows: `ipconfig`
   - Mac/Linux: `ifconfig`
3. على الموبايل، افتح المتصفح واذهب إلى:
   ```
   http://YOUR_IP:5173
   ```
   مثال: `http://192.168.1.100:5173`

## 🎨 تخصيص النظام

### تغيير الألوان
افتح `tailwind.config.js` وعدّل الألوان في قسم `colors`.

### إضافة أسئلة جديدة
افتح `src/data/mockData.js` وأضف أسئلة جديدة في مصفوفة `questions`.

### إضافة جامعات
افتح `src/data/mockData.js` وأضف جامعات في مصفوفة `universities`.

## 📦 البناء للإنتاج

عندما تريد نشر التطبيق:

```bash
npm run build
```

سيتم إنشاء مجلد `dist/` يحتوي على الملفات الجاهزة للنشر.

## 🔑 بيانات تجريبية للاختبار

يمكنك استخدام أي بيانات للتسجيل، النظام لا يتحقق من البريد الإلكتروني.

مثال:
- الاسم: أحمد محمد
- البريد: ahmed@test.com
- كلمة المرور: 123456
- الصف: 11

## ⚠️ ملاحظات مهمة

1. **بيانات مؤقتة**: جميع البيانات محفوظة في المتصفح (localStorage). إذا مسحت بيانات المتصفح، ستفقد حسابك ونتائجك.

2. **لا يوجد Backend**: هذا الإصدار يعمل بدون خادم. جميع العمليات تتم في المتصفح.

3. **نتائج تجريبية**: المعادلات المستخدمة هي نماذج مبسطة. في نظام حقيقي، ستحتاج لخوارزميات أكثر تعقيداً.

4. **للأغراض التعليمية**: هذا مشروع أكاديمي ويجب عدم الاعتماد عليه بشكل كامل لاتخاذ قرارات مصيرية.
